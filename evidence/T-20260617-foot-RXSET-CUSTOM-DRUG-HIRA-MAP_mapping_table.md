## 식별 기준 (AC1)

- '자체' 배지 = `prescription_codes.code_source = 'custom'`
- custom 19종 정확히 식별(기대치 19 = 실측 19 일치). official(EDI/HIRA 마스터) = 499건.
- ⚠ 코드 컬럼 정정: 티켓 추정 `standard_drug_code`/`prescription_code_id` 컬럼은 `prescription_codes`에 **없음**.
  실제 코드 컬럼 = `claim_code`. custom 19종의 현 코드 = `LEGACY-xxxx` 발번값.
- ⚠ custom 19종은 처방세트(`prescription_sets.items`)에 **약이름 문자열로만 19건** 등록, `prescription_code_id`로는 **0건 연결**
  (= RXSET-DRUGSOURCE-SVCRX 입증 '청구코드 손실' 경로와 동일).

## 요약

- **자동확정 가능 = 0종** (AUTO 후보 1건도 제형·성분 불일치=오매칭 → 사람 확인 없이 적용 가능한 건 0).
- MANUAL 9종: 성분 일치하나 official 후보 다수(상품명·용량 상이) → reporter가 실제 처방 상품·용량 1:1 지정 필요.
- NONE 9종: 내부 official 미수록(에피나코나졸·우레아·클로베타솔 등 비급여 신약/전문 외용제) → **외부 HIRA 표준약물 DB 소스 필요**(risk#2).
