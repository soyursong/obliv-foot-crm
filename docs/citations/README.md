# docs/citations — 규정·근거 인용(Citation) 저장소

풋센터 CRM 개발·운영에서 근거로 삼은 **외부 규정·법령·공식 문서의 원문 인용**을 저장·관리하는 곳이다.
"관행"이 아니라 "규정 원문"을 근거로 의사결정했음을 추적 가능하게 남기는 것이 목적이다.

- 발안: 이정환 경영BO ("리모트 레포에 doc 경로 만들어서 규정 citation 저장·관리")
- 신설 티켓: `T-20260714-foot-CITATION-DOC-REPO-STORE`
- 성격: **docs 전용** — 코드/RPC/DB 무변경, 완전 가역

---

## 1. 목적

- CRM 계산 로직(예: 본인부담금 단수처리)·정책 판정의 **근거 조문을 원문 그대로** 보존한다.
- "누가 언제 어떤 URL에서 확인했는지"를 남겨, 나중에 규정이 개정돼도 **당시 근거 vs 현행**을 대조할 수 있게 한다.
- 개발자가 계산식을 짤 때 "왜 이 값/방식인지"를 코드 주석 대신 이 저장소의 citation id로 참조한다.

## 2. 저장 규칙

1. **1건 = 1 md 파일.** 파일명은 citation id를 그대로 쓴다: `CIT-YYYY-NNN-<slug>.md`.
   - `YYYY` = 등록 연도, `NNN` = 그 해의 3자리 일련번호(001부터), `<slug>` = 출처를 알아볼 짧은 영문 슬러그.
2. **원문 인용은 원문 그대로.** 요약·의역 금지. 원문 문구는 인용 블록(`>`)으로 표기한다.
   - 원문을 찾지 못했으면 "미검출"로 정직하게 기록한다. **그럴듯한 조문 날조 금지.**
3. 새 citation을 추가하면 아래 **§4 인덱스** 표에 1행 추가한다.
4. 규정이 개정되어 근거가 바뀌면 기존 파일은 남겨두고(이력 보존) 새 id로 새 파일을 만든다. 기존 파일 상단에 `superseded_by: CIT-...`를 표기한다.
5. 이 저장소는 docs 전용이다. 여기 변경은 코드/DB에 영향을 주지 않는다.

## 3. 표준 포맷 (필수 7필드)

각 citation md는 다음 frontmatter 7필드를 반드시 포함한다.

| 필드 | 설명 |
|------|------|
| `id` | citation 식별자 (`CIT-YYYY-NNN`) |
| `source_org` | 출처 기관 (예: 법제처, 건강보험심사평가원) |
| `document` | 법령·문서명 + 조항 (예: 국민건강보험법 시행령 별표2 제19조 제1항) |
| `quote` | 원문 인용 — **원문 그대로** (본문에 인용 블록으로도 재기재) |
| `url` | 접근 URL |
| `retrieved_at` | 확인 일자 (YYYY-MM-DD) |
| `related_tickets` | 관련 티켓 ID 목록 |

템플릿은 `_TEMPLATE.md` 참조.
