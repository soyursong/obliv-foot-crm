## 2. 대시보드 스크린샷 분석

### 2-1. 코디 대시보드 (60_jongno_dashboard.png)

**관찰 사항:**
- 칸반 보드: 대기 → 상담 3/3 → 시술대기 → 시술 (15실 3열) → 결제대기
- 좌측 사이드바: 오늘 예약 30건 목록, 예약별 체크인/노쇼 버튼
- 시술실 15개가 3열(5행×3열) 구조로 배치 → 한 화면에 전체 파악 가능
- 각 카드: 대기번호 #, 이름, 방금/N분/시간 배지, 마취 타이머(시술가능 표시)
- 상단 요약: 체크인 N, 대기 N, 시술대기 N, 결제대기 N, 완료 N

**발견된 문제:**
1. **날짜 고정 = 오늘만**: 대시보드에 날짜 선택 UI 없음. 어제/과거 데이터 조회 불가
2. **좌측 사이드바 스크롤 과부하**: 예약 30건이 세로 스크롤. 시간대별 그룹핑 없이 전체 나열
3. **시술실 번호와 이름 매핑 없음**: "시술1~15"만 표시. "2층 왼쪽방", "VIP실" 등 실제 방 이름 표시 불가
4. **외국인 구분 약함**: 🌏 이모지만으로 외국인 표시. EN/CN/JP 언어 구분 없음
5. **결제대기 컬럼 좁음**: 대시보드 우측 끝에 배치되어 스크롤 필요

### 2-2. 예약관리 (61_jongno_reservations.png)

**관찰 사항:**
- 주간 캘린더 뷰 (4/6~4/12)
- 상단 요약: 예약 85, 워크인 62, 매출 2421만 (주간 합계)
- 슬롯별 고객명+금액 표시 (e.g., "박지유 100만")
- "W" 마커 = 워크인, "+N명" = 슬롯 오버플로
- 토요일 "~17:00" 표시, 일요일 "휴무" 표시

**발견된 문제:**
1. **드래그 재배치 불가**: 예약 시간 변경 시 삭제→재생성 필요
2. **슬롯당 최대 인원 초과 시각화 미흡**: "+4명" 표시만 있고, 실제 대기 시간 예측 없음
3. **노쇼 이력 표시 없음**: 해당 고객이 과거 노쇼한 적 있는지 예약 단계에서 보이지 않음
4. **시술 소요시간 미표시**: 각 예약이 몇 분 걸리는 시술인지 캘린더에서 안 보임
5. **예약금/선결제 상태 미표시**: 예약에 결제 여부가 보이지 않음

4. **직원별 일정 충돌 검증 없음**: 같은 시간에 같은 선생님이 2실에 배정될 수 있음

## 3. 역할별 불편 포인트 종합

### 코디네이터 (코디)
1. 대기 → 상담 → 시술대기 상태 전환이 드래그 or 버튼 클릭인데, 바쁜 시간에 방 번호 선택까지 해야 함
2. 워크인 고객 수동 등록 시 이름/전화번호 입력 → 고객 검색 → 시술 선택 프로세스가 길다
3. 노쇼 처리 후 복원 시 원래 예약 슬롯이 사라져서 다시 맞추기 어려움
4. 시술실 15실 중 어디가 비었는지 한눈에 안 보임 (실 이름/층 구분 없음)
5. 결제대기 컬럼이 화면 우측 끝에 있어서 놓치기 쉬움

### TM (전화 예약 담당)
1. 예약 생성 시 시술별 소요시간/가격 정보가 바로 안 보여서 별도 참조 필요
2. 기존 고객 검색이 전화번호 기반인데, 전화번호 일부만 기억하는 경우 대응 어려움
3. 리콜 대상(60일+ 미방문) 자동 추출 기능이 없어서 수동 관리 필요
4. 예약 변경(시간 이동) 시 기존 예약 취소→새로 생성해야 해서 2스텝 작업
5. 노쇼 이력이 있는 고객인지 예약 시점에 확인 어려움

### 상담실장/상담사
1. 다음 상담 고객의 정보가 대기 카드에 부족 (시술종류만 표시, 상담 메모/이전 시술 이력 없음)
2. 연속 상담 시 이전 고객 상담 결과를 빠르게 기록할 공간이 없음 (notes 필드만 존재)
3. 본인이 상담한 고객의 매출 집계를 실시간으로 볼 수 없음
4. 상담실 → 시술대기 전환 시 마취 시작 시점을 별도로 체크해야 함
5. 재방문 고객의 이전 시술 사진/결과를 바로 참조할 수 없음

### 시술자 (원장/부원장)
1. 본인에게 배정된 시술실의 대기 환자만 필터링해서 볼 수 없음
2. 마취 타이머가 대시보드에서만 보이고, 시술자 전용 뷰가 없음
3. 시술 완료 후 다음 환자가 자동 안내되지 않음 (코디가 수동 배정)
4. 시술 중 메모(특이사항) 기록이 불편 (대시보드 카드 클릭 → notes 수정)

### 고객
1. 셀프 체크인 후 대기 정보 부족 (몇 번째인지, 예상 대기시간)
2. 대기 화면(waiting screen)에서 진행 상태만 보이고, 시술실 안내 없음
3. 체크인 후 상담실 호출 시 물리적으로 어디로 가야 하는지 안내 없음

## 4. 300명/일 시 코디가 가장 힘든 TOP 10

> 현재 107명/3일 시뮬레이션 기준 추정, 하루 300명으로 스케일업 시

1. **대기열 폭발**: 대기 6명 수준에서도 스크롤 필요 → 300명이면 대기 30명+ 동시. 시간별 그룹핑/자동 호출 순서 필수
2. **시술실 배정 병목**: 15실 × 300명 = 실당 20명/일. 빈 실 찾기가 눈으로는 불가능 → 자동 배정 알고리즘 필요
3. **상담실 회전율 관리**: 3실로 300명이면 실당 100명. 상담 10분 기준 불가능 → 상담실 6실+ 필요 or 사전상담 분리
4. **워크인 처리 지연**: 300명 중 워크인 20%=60명. 수동 등록이 1분씩이면 60분 소요 → 셀프 체크인 의무화 필요
5. **노쇼 관리 혼란**: 300명 중 노쇼 10%=30명. 일일이 체크인/노쇼 처리 → 자동 노쇼 처리(예약시간+15분) 필요
6. **결제대기 누락**: 300명이면 결제대기 20명+ 동시. 우측 끝 컬럼에서 놓치기 쉬움 → 결제대기 알림/미결제 경고 필요
7. **전화 동시 응대**: 예약 문의 + 도착 확인 + 시술 문의가 동시에 → TM과 역할 분리 or 자동 도착 알림 필요
8. **마취 타이머 관리**: 시술대기 30명이면 마취 시작 시점 관리 불가 → 20분 경과 시 자동 알림 + 시술가능 순서 큐 필요
9. **고객 혼선**: 같은 이름 고객이 여러 명일 확률 증가 → 대기번호 기반 호출 시스템 강화 필요
10. **일마감 작업량**: 300건 결제 내역 대사 → 자동 대사 + 이상거래만 하이라이트 필요

## 5. TM이 하루 50건 예약 시 가장 비효율적인 작업 TOP 5

1. **고객 검색 → 신규 등록 반복**: 전화로 이름/번호 확인 → 검색 → 없으면 신규 등록 → 다시 예약 생성. 평균 2분/건 × 50건 = 100분. **개선:** 전화번호 입력 시 자동 검색 + 없으면 즉시 생성 원스텝
2. **시술 가격/소요시간 참조**: 예약 생성 시 시술 선택 드롭다운에 가격만 표시. 소요시간 없어서 슬롯 배정 시 감으로 판단. **개선:** 시술명 옆에 (30분/50만) 표시
3. **리콜 대상 수동 관리**: 60일+ 미방문 고객을 엑셀/별도 시스템에서 추출. **개선:** 고객이력 페이지에 "리콜 대상" 필터 + 원클릭 예약 생성
4. **예약 변경(시간 이동)**: 고객이 시간 변경 요청 시 기존 취소 → 새로 생성. 2건 작업. **개선:** 예약 카드에서 시간만 드래그 or 수정
5. **중복 예약 방지 미흡**: 같은 고객이 같은 날 예약이 있는지 확인하려면 캘린더를 눈으로 스캔. **개선:** 중복 예약 시 경고 팝업

## 6. 상담실장이 연속 상담 시 필요한 정보 누락 TOP 5

1. **이전 시술 이력**: 재방문 고객이 과거에 어떤 시술을 받았는지 대시보드 카드에서 바로 안 보임. 고객이력 탭으로 이동해야 함
2. **고객 메모/특이사항**: 알레르기, 피부 상태, 선호사항 등 상담 메모가 카드에 미표시. 클릭해야 notes 확인 가능
3. **시술 전후 사진**: 이전 시술 결과 사진을 참조할 수 없음 (현재 사진 저장 기능 자체가 없음)
4. **예상 가격/패키지 정보**: 상담 중 고객에게 가격 안내 시, 서비스 목록/가격표를 별도 참조. 상담 카드에서 바로 시술 선택 + 가격 확인 필요
5. **본인 상담 실적**: 오늘/이번 주/이번 달 상담 건수와 전환율(상담→시술 결정)을 실시간으로 볼 수 없음

## 8. 태블릿/모바일 테스트 결과

### 태블릿 (1024×768)
- 대시보드 칸반 보드 전체 표시 가능하나 시술실 15실이 작아짐
- 좌측 예약 사이드바가 좁아져서 정보 잘림
- 예약관리 주간뷰는 잘 표시됨

### 모바일 (390×844)
- 대시보드: 칸반 보드가 가로 스크롤 필요 → 코디가 모바일로 운영하기 어려움
- 셀프 체크인: 정상 동작, 반응형 잘 적용됨
- **결론**: 코디/상담은 태블릿(최소 1024px) 필수. 모바일은 셀프 체크인 전용
