# 대규모 UX 테스트 결과 — 종로 롱래스팅센터 (200~300명/일 시나리오)

> 테스트 일시: 2026-04-10
> 테스트 데이터: 60명 고객, 35건 예약, 25건 체크인 (6 대기 / 3 상담 / 3 시술대기 / 8 시술 / 3 결제대기 / 2 완료), 12명 직원, 17건 방 배정

## TOP 5: 하루 300명 처리 시 코디가 가장 힘든 포인트

### 1. 시술실 15실 한눈에 못 봄 — grid-cols-5 레이아웃 한계 (CRITICAL)
- **현상**: 시술실이 `grid grid-cols-5 gap-1` (112px x 5열)로 배치. 15실이면 3행 필요하지만, 뷰포트 하단에 밀려 스크롤 필요
- **실측**: 시술실 영역 너비 576px, 전체 대시보드가 가로로 대기→상담→시술대기→시술→결제대기 순서로 펼쳐져 있어 시술 영역이 화면 중앙~우측에 위치. 1~10번 방은 보이지만 11~15번은 fold 아래
- **영향**: 코디가 "7번 방 환자 끝났나?"를 확인하려면 가로 스크롤 + 세로 스크롤 필요. 3초 rule 불가
- **제안**: 시술실 그리드를 3열 x 5행 또는 5열 x 3행으로 + 점유/비점유 시각 구분 강화. 빈 방은 회색 축소 표시

### 2. 좌측 예약 사이드바가 끝없이 스크롤됨 (HIGH)
- **현상**: "오늘 예약 30건"이 좌측 사이드바에 1열로 나열. 사이드바 너비 약 145px으로 극히 좁음
- **실측**: 예약 건당 약 60px 높이 → 30건이면 1800px 스크롤 필요 (뷰포트 900px)
- **영향**: 10시 예약자 확인하다가 14시 예약자 보려면 20번 스크롤. "다음에 올 사람"을 즉시 파악 불가
- **제안**: 시간대별 그룹핑 (10:00 ▶ 3명, 10:30 ▶ 5명), 체크인 완료 건은 축소/접기, 현재 시간 기준 하이라이트

### 3. 대기열 카드에 "시술 정보" 없음 (HIGH)
- **현상**: 대기 카드에 대기번호, 이름, 경과시간만 표시. 어떤 시술 예정인지 안 보임
- **실측**: 대기 #004 윤서현 — "방금" 표시만 있고, 헤어라인인지 엠보인지 모름
- **영향**: 코디가 "다음 상담실에 누구 넣을까" 판단 시 카드만 보고는 불가 → 별도 예약 정보 확인 필요 (더블 클릭? 컨텍스트 메뉴?)
- **제안**: 카드에 시술명 1줄 추가 (예: "헤어라인(풀)" 또는 "엠보+점막")

### 4. 체크인/노쇼 버튼 터치 타겟 너무 작음 (HIGH)
- **현상**: 예약 카드의 "체크인" 버튼 높이 20px, "노쇼" 버튼 42x20px
- **실측**: Apple HIG 최소 44x44px, WCAG 최소 24x24px 기준 모두 미달
- **영향**: 태블릿 사용 시 잘못된 터치 빈번. 바쁜 시간대에 노쇼를 체크인으로 잘못 누를 위험
- **제안**: 최소 높이 36px, 모바일/태블릿에서는 44px

### 5. 결제대기 컬럼이 화면 끝에 잘림 (MEDIUM)
- **현상**: 결제대기 컬럼이 화면 맨 우측에 위치. 1440px에서도 "#007 오예진" 카드가 우측 잘림 ("4시..." 텍스트 truncate)
- **실측**: 결제대기 1건만 표시. 3~5건일 때 세로 스크롤 필요
- **영향**: 데스크 직원이 결제대기 환자를 놓칠 수 있음
- **제안**: 결제대기는 상단 고정 배너 또는 토스트로 알림. 또는 대시보드 우측 고정

## 시나리오별 상세 분석

### 시나리오 1: 오전 피크 (체크인 25, 대기 6, 상담 3/3, 시술대기 3, 시술 8/15, 결제대기 3)
- **스크린샷**: `deep_a_jongno_selected.png`
- 대시보드 상단 요약: "체크인 25 대기 6 시술대기 3 결제대기 3 완료 2" — 숫자는 명확하고 한눈에 들어옴 ✅
- 좌측 예약 사이드바: 30건 나열, 시간순 정렬 ✅, 하지만 스크롤 과다 ❌
- 상담실: "상담 1 (2/2)", "상담 2", "상담 3" — 방 번호와 수용 인원 표시 ✅
- 시술실: 1~10번 방에 환자 카드, 시술자 이름, 마취 시간, "시술가능" 태그 표시 ✅
- 결제대기: 우측 끝에 3건 (#021, #022, #023) — 경과 시간 표시 ✅, 위치가 눈에 안 들어옴 ❌

**코디 관점 "3초 rule" 테스트:**
- "지금 대기 몇 명?" → 상단 숫자 즉시 확인 ✅ (1초)
- "다음 상담실 비는 곳?" → 상담 영역 확인 가능 ✅ (2초)
- "시술 7번 방 환자 누구?" → 시술 그리드에서 7번 찾기 필요, 화면 내 보이면 3초, 스크롤 필요하면 5초+ ⚠️
- "결제 대기 누구?" → 화면 우측 끝 확인 필요, 3초 이상 ❌

### 시나리오 2: 점심 피크 대시보드
- 송도점 기준 (초기 선택): 체크인 19, 대기 6, 시술대기 2, 결제대기 1, 완료 5
- 스크롤 없이 대시보드 전체 확인 가능 (bodyHeight 904px ≈ viewport 900px) ✅
- 하지만 종로점 선택 시 데이터 양 증가로 세로 스크롤 발생 ⚠️

### 시나리오 3: TM 동시 예약
- **스크린샷**: `s3_tm1_new_reservation_modal.png` (예약 관리 주간 캘린더)
- 두 TM이 동시에 예약 관리 페이지 접속 시 동일한 데이터 표시 ✅
- 예약 43건, 워크인 24건, 매출 140만 (송도) / 예약 53건, 워크인 35건, 매출 967만 (종로)
- **문제**: 같은 시간대 잔여 슬롯 수 표시 없음. "+3명" 같은 축약 표시는 있지만 "최대 몇 명까지 가능한지" 기준 없음 ❌
- **제안**: 시간대별 슬롯 수용 한도 vs 현재 예약 수 시각화 (예: "10:00 — 5/8명")

### 시나리오 4: 상담실장 연속 상담
- 카드 클릭 시 반응: 카드 클릭해도 모달/패널 안 열림 (context menu만 작동해야 하는 구조)
- **우클릭 컨텍스트 메뉴**: Playwright headless에서 메뉴 아이템 감지 안 됨 (DOM에 role="menuitem" 없음)
- 상담→시술대기 상태 전환: 드래그&드롭 또는 컨텍스트 메뉴 방식으로 추정
- **측정**: 5명 연속 처리 시 카드 클릭 → 상태 변경 → 다음 환자 = 최소 3클릭 × 5명 = 15클릭
- **제안**: "다음 단계로" 원클릭 버튼을 카드에 직접 노출

### 시나리오 5: 시술실 피크
- **스크린샷**: `deep_a_jongno_selected.png`
- 시술실 그리드: `grid grid-cols-5 gap-1` — 5열 × 3행 (15실)
- 방 1~10에 환자 배치 확인 ✅
- 빈 방(11~15)은 빈 셀로 표시 — 시각적으로 "비어있음"이 명확하지 않음 ❌
- 마취 시간 표시: "마취 -531분" 같은 음수 표시 → 마취 경과시간 음수는 의미 혼동 ❌
- 시술자 이름 + "시술가능" 태그: 시술 상태 파악에 유용 ✅
- **시술자 관점**: 자기 방(7번) 환자를 찾는데 그리드에서 위치 파악 필요. 방 번호 레이블이 작음 (12px 미만)

### 시나리오 6: 데스크 결제 러시
- 결제대기 3건: #021 강다영, #022 이유진, #023 박서연
- 각 카드에 경과시간 표시 ("마취 -647분" 등 — 이 숫자 의미 불명)
- **문제**: 결제 금액이 카드에 미표시. 데스크 직원이 결제 시작 전에 금액 모름 ❌
- **제안**: 결제대기 카드에 시술 금액 합계 표시

### 시나리오 7: 고객 대기 화면
- 체크인 폼: 이름, 전화번호(국가코드 포함), 방문 경로(네이버/인스타/지인/블로그/유튜브/네이버지도/기타), 개인정보 동의 ✅
- 폼 UI 깔끔하고 직관적 ✅
- **문제**: input name이 "name"/"phone"이 아닌 다른 형식이라 자동화 테스트에서 필드 접근 어려움 (placeholder 기반)

### 시나리오 8: 태블릿 1024x768

## UI/UX 디자이너 관점 종합 분석

### 정보 계층 (Information Hierarchy)
- ✅ 상단 요약 바 ("체크인 25 대기 6...") — 가장 중요한 정보가 최상단
- ✅ 컬럼별 상태 구분 (대기/상담/시술대기/시술/결제대기) — 칸반 보드 패턴
- ❌ 시술실 내 방 번호 계층이 약함 — 방 번호 < 환자명 순서여야 하는데 동일 비중
- ❌ 예약 사이드바의 시간대 구분 없음 — 30건이 flat list

### 색상 대비 (Color Contrast)
- ✅ 상태별 색상 코드 명확: 대기(회색), 상담(파랑), 시술대기(노랑), 시술(초록), 결제대기(보라)
- ✅ 노쇼 카드 핑크/빨강 배경 — 눈에 잘 띔
- ⚠️ "방금" 태그 (주황) vs 경과시간 (빨강) — 색상만으로 의미 구분 어려움
- ❌ 마취 시간 "-463분" 녹색 텍스트 — 음수인데 녹색이라 혼동

### 터치 타겟 (Touch Targets)
- ❌ "체크인" 버튼: 115x20px (높이 20px — 최소 기준 44px의 45%)
- ❌ "노쇼" 버튼: 42x20px (높이 20px)
- ❌ "관리" 버튼: 43x24px
- ⚠️ 예약 카드 자체는 충분한 크기 (약 145x60px)
- **결론**: 태블릿 환경에서 버튼 오터치 위험 높음

### 폰트 크기
- ❌ 예약 카드 내 "예약" 라벨: 8px — 가독성 심각
- ❌ referral source (naver, walk_in 등): 8px
- ❌ "체크인"/"노쇼" 버튼 텍스트: 9px
- ⚠️ 고객명 (사이드바): 10px — 가독성 한계
- ✅ 대기번호 (#004): 14px+
- ✅ 상단 요약 숫자: 충분한 크기

### 여백 (Spacing)
- ✅ 컬럼 간 간격 적절
- ❌ 예약 사이드바 카드 간 간격 4px — 너무 촘촘
- ❌ 시술실 그리드 gap-1 (4px) — 15실이 빽빽하게 나열

### 인터랙션 피드백
- ⚠️ 카드 클릭 시 시각적 피드백 없음 (선택 상태 표시 없음)
- ⚠️ 우클릭 컨텍스트 메뉴 존재하지만 접근성 이슈 (태블릿에서 우클릭 불가)
- ❌ 드래그&드롭 힌트 없음 — 카드가 드래그 가능하다는 시각적 단서 없음
